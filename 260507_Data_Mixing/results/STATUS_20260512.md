## OOM Incidents and Fixes

| Attempt               | Setting                                    | Result                |
| --------------------- | ------------------------------------------ | --------------------- |
| 5090_Lian first       | `gpu_mem=0.45, max_num_seqs=16, 32k`       | OOM after 14 prompts  |
| 5090_Lian second      | `gpu_mem=0.90, max_num_seqs=8, 32k`        | OOM immediately (KV over-allocated, no Triton workspace headroom) |
| 5090_Lian third       | `gpu_mem=0.55, no max_model_len, max_num_seqs=4` | Ran, but ETA 24–48h |
| 5A100 current (×4)    | `gpu_mem=0.45, max_num_seqs=32, 32k`       | Stable, ETA ~9–13h    |

Root cause: Triton workspace buffers are outside vLLM's predicted memory budget on the 5090s; the A100s have enough headroom with `gpu_mem=0.45` and eager-mode.
